## 一、相对 v2，新增/变化了什么

v2 已经讲了：①补 Omada 自身舆情板块 ②impact badge 分语义 ③用户舆情改名竞品舆情。**那些仍然有效，继续做。**

本轮（v3）在此基础上再加两点：

### 🆕 变化 1：周报新增「🎯 市场策略洞察」区（需要新设计）— 最重要

**这是 v2 没有的全新区块，只在周报出现。**

周报的板块顺序变成（注意策略洞察置顶）：
```
0. 🎯 市场策略洞察   ← 全新，置顶，战略级
1. 🟢 Omada 自身舆情
2. ⚔️ 竞品动态
3. 🗣️ 竞品舆情与对比
4. 🏪 store 动向
5. 🏭 行业风向
6. 📊 数据看板
```

**它长什么样**（设计要点）：
- **不是卡片流，是一个「洞察区块」**——以文字论述为主，比普通情报卡片更「重」（更宽 / 更高对比度 / 可加边框或浅背景区分）
- 顶部标「🎯 市场策略洞察 · 本周」+「OPUS 策展」来源徽章（和日报导语一致的策展标识）
- 主体是 2~4 段战略判断文字：市场格局 → 竞品战略意图 → 对 Omada 的中长期建议
- **结论里嵌可点击上标溯源** `¹ ⁴ ⁵`，或底部「依据：[1] [4] [5]」——战略判断同样不允许无源，点击跳到对应事件卡片
- 视觉气质：像券商研报的「核心观点」框 / Stratechery 的开篇论述区

**为什么加**：之前的产品全是「事件级」（今天谁发了新品/谁在抱怨）。这个区是「战略级」——Opus 读一周的竞品+舆情+财报，输出"市场怎么走、Omada 该怎么应对"。这是把「事件聚合器」升级成「情报站」的关键一区。

样例文案见 `docs/SAMPLE_REPORT.md` 末尾「周报片段样例 — 市场策略洞察」。

---

### 🆕 变化 2：数据源标注调整（轻微，影响来源徽章文案）

舆情数据源从「个人情报流」改成专用系统 **omada-sentiment-monitor**。对设计的影响很小，只是：

- 舆情卡片的来源副标可以体现「带情感分析」——例如卡片上可显示 `情感:中性` / `相关性:0.9` / `切换意图` 这类标签（数据里现在有这些 AI 字段）
- 来源徽章逻辑不变：还是按 reddit.com / youtube.com / community.ui.com / blog.ui.com 区分一手/二手

**可选增强**：舆情卡片可以多一个小标签位显示 `情感倾向`（正面/中性/负面），因为新数据源带这个字段。设计上可作为卡片的一个 meta 标签，不强制。

## 二、JSON 契约变化（给前端 + 设计参考）

`report.json` 新增一个顶层字段 `strategy`（仅周报有，日报为 null）：

```json
{
  "report_id": "2026-W22-weekly",
  "type": "weekly",
  "strategy": {
    "title": "市场策略洞察",
    "body": "本周 UniFi 在 5G/WAN 冗余赛道集中投入{{cite:1}}{{cite:4}}……Omada 建议在蜂窝备份与多运营商控制上差异化{{cite:5}}。",
    "cite_refs": [1, 4, 5]
  },
  "lead": { ... },
  "sections": [ ... ],
  "items": [ ... ],
  "references": [ ... ],
  "stats": { ... }
}
```

- `strategy.body` 里的 `{{cite:N}}` 占位符 → 渲成可点上标（和 `lead` 一样的处理）
- 日报 `strategy: null`，前端不渲染该区
- 舆情类 item 现在可能带额外字段：`sentiment`（正面/中性/负面）、`relevance`（0~1）、`switch_intent`（bool，流失意图）——设计可选用这些做 meta 标签

完整 Schema 见 `docs/PRD.md §7.9`。
