# 输出格式

下面是一段联网检索得到的**行业研究记录**，以及它引用的**来源列表（SOURCES）**（每条带编号）。请把研究记录整理成结构化 JSON 数组，供周报「行业风向」板块使用。

每个条目字段：
- `title`：信号标题（中文优先，可保留必要英文术语）
- `source_index`：该信号对应的来源**编号**（整数，**必须来自下面的 SOURCES 列表**）。你**不要写 URL**，只给编号；找不到对应来源的信号**直接丢弃**。
- `date`：原文发布日期 `YYYY-MM-DD`；确实不可知时留空
- `summary`：一句话中文摘要，点明事实 + 对 Omada 的意义

规则：
- 只保留能**对应到 SOURCES 真实来源**的条目；宁缺毋滥。
- 同一事件**去重**，合并为一条。
- 保留最有价值的若干条（约 3–8 条），不要凑数。
- 只输出 JSON 数组，不要多余文字。
